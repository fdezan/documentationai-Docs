---

## Service Call e Condition Types por Imposto

A condition type **TXSC** dispara a chamada HTTP. As demais leem do buffer.
Cada imposto tem **multiplas** condition types (mapeamento via J_1BNFTXCOND).

| Imposto | Condition Types (KSCHL) | Fonte |
|---------|------------------------|-------|
| **Service Call** | TXSC | `if_txs_constants.abap:146` |
| **ICMS** | BX16, ICM1, ICM2, BX10, BX11, BX12, BIC1 | `if_logbr_ft_txs_constants.abap:93-99` |
| **IPI** | BIP0, IPI1, IPI2, BX20, BX21, BX22, BXPA, BIP1 | `:101-108` |
| **PIS** | BPI1, IPIS, IPSN, BX80, BX81, BX82, BPIP | `:110-116` |
| **COFINS** | BCO1, ICOF, ICON, BX70, BX71, BX72, BCOP | `:118-124` |
| **II** | BXIR, II01, BASB, BXIE, BXII | `:126-130` |
| **IRRF** | BW61, BX9U, BX9V | `if_txs_constants_br.abap:110-112` |
| **INSS** | BW71, BX9R, BX9S | `:113-115` |

<Callout kind="info">
ISS, CSLL, CBS: condition types nao confirmadas no codigo — validar no sistema SAP.
</Callout>

## Tabelas de Customizing

| Tabela | Funcao | Campos-Chave |
|--------|--------|-------------|
| TXS_ACTIVATION | Ativa TXS por cenario | KAPPL + KALSM + ACTIVE |
| TXS_ACTIVATION_EXCEPTION | Excecoes empresa/centro | BUKRS + WERKS |
| TXS_TAX_CODE_MAP | Tax code externo → S/4 | COUNTRY + APP + PROCEDURE + EXT_CODE → MWSKZ |
| TXS_TAX_CODE_MAP_C | Variante custom (prioridade) | Mesma estrutura |
| TXS_COND_MAP | Mapper de condition type | CONDITION_TYPE + CLASS |
| TXS_COND_MAP_C | Variante custom (prioridade) | Mesma estrutura |

**KAPPL**: TX = MM, V = SD

## Mapeamento Resposta → KOMV

| Resposta Sovos | KOMV | Formula |
|---------------|------|---------|
| value | KWERT | Valor direto |
| rate | KBETR | rate x 10 |
| taxable | KAWRT | Valor direto |
| exempted_base_percent | KBETR | rate x (100 - exempt%) / 100 x 10 |
| (sem match) | KINAK = 'X' | Condicao inativa |
