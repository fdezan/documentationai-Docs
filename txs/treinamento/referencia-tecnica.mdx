## 1. Arquitetura — As 8 Fases do TXS

O Pricing SAP aciona o TXS via **Formula 279**. Uma unica chamada HTTP ao motor externo calcula todos os impostos; as demais conditions leem do cache (Response Buffer).

| Fase | Nome | Classe Principal | Metodo de Entrada |
|------|------|-----------------|-------------------|
| 1 | Entrada e Validacoes | CL_TXS_CONTROLLER_BR | execute() |
| 2 | Decisao (Service Call ou Mapping) | CL_TXS_CONTROLLER | process_condition_record() |
| 3 | Build do Payload JSON | CL_TXS_PAYLOAD_FACTORY | get_payload() |
| 4 | Comunicacao HTTP (OAuth2) | CL_TXS_COMMUNICATION | calculate_taxes() |
| 5 | Processamento da Resposta | CL_TXS_RESPONSE_BR | constructor() |
| 6 | Mapeamento para KOMV | CL_TXS_CONDITIONS_MAPPER_BR | map_tax_type_to_condition() |
| 7 | Verificacao de Mapeamento | CL_TXS_CONDITION_PROCESSING_BR | check_mapping() |
| 8 | Pos-Processamento e Erros | CL_TXS_CONTROLLER | error_handling() / post_processing() |

### 2.2 Troubleshooting — 10 Cenarios

| # | Problema | Classe → Metodo | Verificar | Solucao Tipica |
|---|---------|----------------|-----------|----------------|
| 1 | TXS nao chamado (impostos zerados) | CL_TXS_SWITCH → IS_ACTIVE | Formula 279 no procedure? TXS_ACTIVATION? Exception? | Criar entrada TXS_ACTIVATION ou remover excecao |
| 2 | ICMS zero ou aliquota errada | CL_TXS_CONDITIONS_MAPPER_BR → MAP_TAX_TYPE_TO_CONDITION | tax_type ICMS na response? exempted_base_percent? | Corrigir dados de entrada (UF, NCM, contribuinte) |
| 3 | "mapping_condition_error" | CL_TXS_CONDITION_PROCESSING_BR → CHECK_MAPPING | Todas conditions com KINAK='X'? | Verificar condition types na pricing procedure |
| 4 | Tax code invalido (CX_TXS_TAX_CODE_CONFIG_ERROR) | CL_TXS_TAX_CODE_DAO → GET_S4_TAX_CODE_SINGLE | 0 matches ou N matches na TXS_TAX_CODE_MAP | Adicionar/corrigir entrada na tabela |
| 5 | HTTP timeout / erro conexao | CL_TXS_CONNECTION → GET_REST_CLIENT | OAuth2 OK? Logical port ativo? Firewall? | Verificar Communication Arrangement |
| 6 | NCM vazio no payload | CL_TXS_ITEM_CLASSIFICATIONS_BR → GET_MATERIAL_NCM | MARC-STEUC preenchido? 8 digitos? | Preencher STEUC no mestre de material |
| 7 | CNAE vazio no payload | CL_TXS_CNAE_BR → cnae_for_*() | BRF+ configurado? Fallback preenchido? | Configurar BRF+ ou preencher LFA1/KNA1-CNAE |
| 8 | Payload V1 quando deveria ser V2 | CL_TXS_PAYLOAD_TYPE → DETERMINE_PAYLOAD_TYPE | SD + integracao V2 ativa? (Brasil usa V1) | Verificar ativacao V2 |
| 9 | Buffer vazio na 2a condition | CL_TXS_RESPONSE_BUFFER → GET_RESPONSE | 1a condition (TXSC) executou com sucesso? | Resolver erro da TXSC primeiro |
| 10 | Impostos calculados mas KOMV vazio | CL_TXS_COND_MAPPER_FACTORY → INITIALIZE | Mapper instanciado? TXS_COND_MAP interceptando? | Verificar classe na TXS_COND_MAP |

## 10. Mapeamento Reverso — Response do Motor → KOMV

### 10.1 Campos da Resposta → KOMV

| Campo da Resposta JSON | Campo KOMV | Logica | Exemplo |
|----------------------|-----------|--------|---------|
| value | **KWERT** | Valor do imposto | 1800.00 |
| rate x 10 | **KBETR** | Aliquota multiplicada por 10 | 18% → **180** |
| taxable | **KAWRT** | Base de calculo | 10000.00 |
| exempted_base_percent | **KBETR** (reduzido) | rate x (100 - exempt%) / 100 x 10 | 33.33% → KBETR = 120 |
| exempted_base_amount | **KWERT** (exclusao) | Valor da base isenta | — |
| other_base_amount | **KWERT** (pauta) | Pauta fiscal / base alternativa | — |
| (sem match) | **KINAK** = 'X' | Condition inativada | Imposto nao se aplica |
| taxcode | **MWSKZ** | Tabela TXS_TAX_CODE_MAP: externo → S/4 | BR_ICMS_18 → I1 |

Classe mapper: `CL_TXS_CONDITIONS_MAPPER_BR`
Classe tax code: `CL_TXS_TAX_CODE_DAO`

### 10.2 Service Call e Condition Types por Imposto

A condition type que dispara a chamada HTTP ao tax engine e **TXSC** (Fonte: `if_txs_constants.abap:146`).
Todas as demais conditions leem do Response Buffer (singleton). Todas usam **Formula 279**. **TXSC** deve ser a **primeira** condition type na pricing procedure.

Cada imposto possui **multiplas** condition types (nao apenas uma). O mapeamento entre condition type e campo da KOMV e feito via tabela **J_1BNFTXCOND**.

Fonte: `if_logbr_ft_txs_constants.abap:91-131` (constante `tax_conditions`)

| Imposto | Condition Types (KSCHL) | Fonte |
|---------|------------------------|-------|
| **Service Call** | TXSC | `if_txs_constants.abap:146` |
| **ICMS** | BX16, ICM1, ICM2, BX10, BX11, BX12, BIC1 | `if_logbr_ft_txs_constants.abap:93-99` |
| **IPI** | BIP0, IPI1, IPI2, BX20, BX21, BX22, BXPA, BIP1 | `if_logbr_ft_txs_constants.abap:101-108` |
| **PIS** | BPI1, IPIS, IPSN, BX80, BX81, BX82, BPIP | `if_logbr_ft_txs_constants.abap:110-116` |
| **COFINS** | BCO1, ICOF, ICON, BX70, BX71, BX72, BCOP | `if_logbr_ft_txs_constants.abap:118-124` |
| **II** | BXIR, II01, BASB, BXIE, BXII | `if_logbr_ft_txs_constants.abap:126-130` |

Condition types adicionais (Fonte: `if_txs_constants_br.abap:107-120`):

| Imposto | Condition Types (KSCHL) | Funcao |
|---------|------------------------|--------|
| **PIS** (pauta) | BPI8 | Pauta (valor) |
| **COFINS** (pauta) | BCO8 | Pauta (valor) |
| **IRRF** | BW61, BX9U, BX9V | WHT collection, base, rate |
| **INSS** | BW71, BX9R, BX9S | WHT collection, rate, base |
| **ICMS** (Zona Franca) | BXRF, BXBF | Rate, base |
| **PIS** (fornecedor) | CPIS | Fornecedor |
| **COFINS** (fornecedor) | CCOS | Fornecedor |

<Callout kind="warning">
**Nota**: Condition types especificas para ISS, CSLL e CBS nao foram encontradas na constante `tax_conditions` nem em `condition_types_values`. O mapeamento desses impostos pode usar conditions genericas ou ser configurado via tabela J_1BNFTXCOND. ⚠ Validar no sistema SAP.
</Callout>
