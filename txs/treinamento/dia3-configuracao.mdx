## Treinamento: Tax Service — Motor de Calculo Externo (Sovos)
**Publico**: Consultores SAP MM/SD Pleno/Senior

---

## 1. Ativacao do TXS

### Tabela TXS_ACTIVATION — Liga/Desliga o TXS

| Campo | Tipo | Descricao | Exemplo |
|-------|------|-----------|---------|
| KAPPL | Char 2 | Aplicacao | **TX** = MM, **V** = SD |
| KALSM | Char 6 | Pricing Procedure | TXBRA1 |
| ACTIVE | Flag | Ativo? | X = Sim |

**Leitura**: "O TXS esta ativo para a aplicacao TX (MM) com o pricing procedure TXBRA1?"

Para ativar TXS em MM e SD, sao necessarias **2 entradas**:

| KAPPL | KALSM | ACTIVE | Significado |
|-------|-------|--------|-------------|
| TX | TXBRA1 | X | TXS ativo para MM com TXBRA1 |
| V | TXBRA1 | X | TXS ativo para SD com TXBRA1 |

### Tabela TXS_ACTIVATION_EXCEPTION — Excecoes

| Campo | Tipo | Descricao | Exemplo |
|-------|------|-----------|---------|
| BUKRS | Char 4 | Empresa | 1000 |
| WERKS | Char 4 | Centro | BR01 |
| ACTIVE | Flag | Ativo/Inativo | ' ' = Desativado |

**Uso**: Desativar TXS para uma empresa ou centro especifico, mesmo que a ativacao geral esteja ON.

**Exemplo**: TXS ativo para TXBRA1, mas desativado para o centro BR99 (piloto):

| BUKRS | WERKS | ACTIVE |
|-------|-------|--------|
| 1000 | BR99 | ' ' |

### Verificacao de Ativacao — 5 Passos Sequenciais

A classe `CL_TXS_SWITCH::IS_ACTIVE()` verifica na seguinte ordem:

1. Conexao ativa? (Logical Port + HTTP)
2. Ativacao por aplicacao + procedure? (TXS_ACTIVATION)
3. Excecao por empresa/centro? (TXS_ACTIVATION_EXCEPTION)
4. Controle V2? (para SD com payload V2)
5. Resultado final: `r_active = true/false`

## 2. Pricing Procedure — Condition Types com Formula 279

### Estrutura Tipica de um Pricing Procedure para TXS

| Step | Condition Type | Descricao | Formula | Notas |
|------|---------------|-----------|---------|-------|
| ... | ... | (outras condicoes de preco) | ... | |
| 500 | **TXSC** | Service Call | **279** | FAZ a chamada HTTP ao Sovos |
| 510 | **BX16** | ICMS (1 de 7) | **279** | Mapeia ICMS do buffer |
| 520 | **ICM1** | ICMS (2 de 7) | **279** | Mapeia ICMS do buffer |
| 530 | **BIP0** | IPI (1 de 8) | **279** | Mapeia IPI do buffer |
| 540 | **BPI1** | PIS (1 de 7) | **279** | Mapeia PIS do buffer |
| 550 | **BCO1** | COFINS (1 de 7) | **279** | Mapeia COFINS do buffer |
| ... | ... | (demais conditions por imposto) | **279** | Configuradas via J_1BNFTXCOND |

**Regra critica**: **TXSC** deve ser a **primeira** condition type com Formula 279 — e ela que faz o HTTP call e armazena a resposta no buffer. Cada imposto tem **multiplas** condition types (ex: ICMS tem 7).

## 3. Conexao HTTP — Logical Port + OAuth2

### Componentes da Conexao

| Componente | Descricao | Onde Configurar |
|-----------|-----------|-----------------|
| **Logical Port** | Porta logica apontando para o endpoint Sovos | Communication Arrangement |
| **OAuth2 Client** | Client ID + Client Secret | Communication System |
| **URL Endpoint** | URL da API do Sovos | Communication Arrangement |
| **Certificate** | Certificado SSL (se necessario) | Trust Management |

### Fluxo de Autenticacao

```
SAP                                  Sovos
 |                                     |
 |--- 1. Request OAuth2 Token -------->|
 |    (client_id + client_secret)      |
 |                                     |
 |<-- 2. Access Token (Bearer) --------|
 |                                     |
 |--- 3. POST /tax/calculate -------->|
 |    (Authorization: Bearer token)    |
 |    (Content-Type: application/json) |
 |    (X-CorrelationID: UUID)          |
 |    (Body: payload JSON)             |
 |                                     |
 |<-- 4. Response JSON ----------------|
 |    (tax_lines, tax_values)          |
```

### Feature Toggles para Debug

| Toggle | Descricao | Quando Ativar |
|--------|-----------|---------------|
| GLO_LOG_BR_EXT_JSON_RESP | Registra JSON de resposta no log | Sempre em desenvolvimento/teste |
| TXS_ACTIVATION | Liga/desliga TXS por cenario | Configuracao inicial |
| TXS_ACTIVATION_EXCEPTION | Excecoes por empresa/centro | Rollout gradual |

## 4. Mapeamento de Tax Codes — TXS_TAX_CODE_MAP

### Para Que Serve?

O Sovos retorna um **tax code externo** (ex: `BR_ICMS_18`). O SAP precisa saber qual **tax code S/4** (ex: `I1`) corresponde.

### Estrutura da Tabela

| Campo | Tipo | Descricao | Exemplo MM | Exemplo SD |
|-------|------|-----------|-----------|-----------|
| COUNTRY | Char 2 | Pais | BR | BR |
| COND_USAGE | Char 1 | Uso da condicao | A | A |
| APPLICATION | Char 2 | Aplicacao | **TX** | **V** |
| PRICING_NAME | Char 6 | Pricing Procedure | TXBRA1 | TXBRA1 |
| MWSKZ | Char 2 | Tax code S/4 | I1 | I1 |
| EXTERNAL_TAX_CODE | Char | Tax code Sovos | BR_ICMS_18 | BR_ICMS_18 |
| TAX_JURISDICTION_CODE | Char | Jurisdicao (opcional) | 3550308 | |

### Variantes

| Tabela | Prioridade | Uso |
|--------|-----------|-----|
| **TXS_TAX_CODE_MAP_C** | Alta (customizada) | Manutencao do cliente |
| **TXS_TAX_CODE_MAP** | Normal (standard) | Entregue pela SAP/parceiro |

**Regra**: A variante customizada (_C) sempre tem prioridade.

### Erros Comuns

| Erro | Causa | Solucao |
|------|-------|---------|
| **0 matches** → `CX_TXS_TAX_CODE_CONFIG_ERROR` | Tax code do Sovos nao existe na tabela | Adicionar entrada na TXS_TAX_CODE_MAP |
| **N matches** → `CX_TXS_TAX_CODE_CONFIG_ERROR` | Multiplas entradas para o mesmo tax code externo | Remover duplicatas ou especificar jurisdicao |

## 5. Mapeamento de Condition Types — TXS_COND_MAP

### Para Que Serve?

Define qual **classe ABAP** faz o mapeamento de cada condition type. Normalmente o standard ja funciona, mas voce pode customizar.

### Estrutura da Tabela

| Campo | Tipo | Descricao | Exemplo |
|-------|------|-----------|---------|
| COUNTRY | Char 2 | Pais | BR |
| COND_USAGE | Char 1 | Uso da condicao | A |
| APPLICATION | Char 2 | Aplicacao | TX |
| PRICING_NAME | Char 6 | Pricing Procedure | TXBRA1 |
| CONDITION_TYPE | Char 4 | Condition type | BX16 |
| CLASS | Char 30 | Classe de mapeamento | CL_TXS_CONDITIONS_MAPPER_BR |

### Factory de Mapeamento — 3 Niveis

A classe `CL_TXS_COND_MAPPER_FACTORY` decide qual mapper usar:

```
1. Classe customizada em TXS_COND_MAP? → Usa essa classe
   |
   v  (se nao encontrou)
2. Payload V2 ativo? → CL_TXS_STANDARD_TAX_MAP_V2
   |
   v  (se nao e V2)
3. V1 Brasil? → CL_TXS_CONDITIONS_MAPPER_BR
   |
   v  (fallback final)
4. Base → CL_TXS_STANDARD_TAX_MAP
```

## 6. Como a Resposta do Sovos Vira KOMV

### Estrutura da Resposta JSON

```json
{
  "id": "uuid-da-chamada",
  "total": 11800.00,
  "sub_total": 10000.00,
  "total_tax": 1800.00,
  "tax_lines": [
    {
      "tax_values": [
        {
          "tax_type_code": "ICMS",
          "rate": 18.0,
          "taxable": 10000.00,
          "value": 1800.00,
          "exempted_base_percent": 0.0,
          "other_base_amount": 0.0
        },
        {
          "tax_type_code": "PIS",
          "rate": 1.65,
          "taxable": 10000.00,
          "value": 165.00
        },
        {
          "tax_type_code": "COFINS",
          "rate": 7.6,
          "taxable": 10000.00,
          "value": 760.00
        }
      ]
    }
  ]
}
```

### Mapeamento Resposta → KOMV

| Campo da Resposta | Campo KOMV | Logica de Mapeamento | Exemplo |
|-------------------|------------|---------------------|---------|
| `value` | **KWERT** | Valor do imposto calculado | 1800.00 |
| `rate` x 10 | **KBETR** | Taxa multiplicada por 10 | 18% → **180** |
| `taxable` | **KAWRT** | Base de calculo | 10000.00 |
| `exempted_base_percent` | **KBETR** (reduzido) | Se > 0: KBETR = rate × (100 - exempt%) / 100 × 10 | Se 33.33%: 18 × 0.6667 × 10 = **120** |
| `other_base_amount` | **KWERT** | Pauta fiscal / tarifa alternativa | Valor da pauta |
| (sem match para tax_type) | **KINAK** = 'X' | Condicao desativada | Imposto nao se aplica |

### Fluxo de Processamento

```
Resposta JSON do Sovos
    |
    v
CL_TXS_JSON_SERIALIZER::deserialize()
    |  Converte JSON → estrutura TXS_RESPONSE
    v
CL_TXS_RESPONSE_BR
    |  Instancia resposta com dados BR
    v
CL_TXS_TAX_TYPES_VALUES_BR
    |  Para cada tax_line:
    |    BUILD_MM_ICMS() / BUILD_SD_ICMS()
    |    BUILD_MM_IPI() / BUILD_SD_IPI()
    |    BUILD_MM_PIS() / BUILD_SD_PIS()
    |    BUILD_MM_COFINS() / BUILD_SD_COFINS()
    |    BUILD_MM_ISS() / BUILD_MM_IR() / BUILD_MM_CSLL()
    v
MAPPED_TAX_TYPES[] (array de impostos parseados)
    |
    v
CL_TXS_CONDITIONS_MAPPER_BR
    |  Para cada condition type (BX16, BIP0, BPI1, BCO1...):
    |    1. Encontra tax_type correspondente (ICMS, IPI, PIS...)
    |    2. Se encontrou: preenche KWERT, KBETR, KAWRT
    |    3. Se nao encontrou: KINAK = 'X'
    v
KOMV preenchida
    |
    v
CL_TXS_TAX_CODE_DAO
    |  Busca TXS_TAX_CODE_MAP:
    |    external_tax_code → MWSKZ (tax code S/4)
    v
Pricing recebe KOMV com todos os impostos
```

## 7. Exercicio: Monte a Configuracao

### Cenario

Uma empresa esta implementando S/4HANA Cloud com TXS + Sovos. Dados:

- **Empresa**: 1000
- **Centros**: BR01 (SP), BR02 (MG), BR03 (RJ)
- **Pricing Procedure**: TXBRA1
- **Modulos**: MM e SD

O Sovos retorna os seguintes tax codes externos:

| Tax Code Sovos | Significado |
|---------------|-------------|
| BR_ICMS_18 | ICMS 18% (operacao interna SP) |
| BR_ICMS_12 | ICMS 12% (interestadual SP→MG) |
| BR_ICMS_7 | ICMS 7% (interestadual para N/NE/CO) |
| BR_IPI_5 | IPI 5% |
| BR_IPI_0 | IPI isento |
| BR_PIS_165 | PIS 1.65% |
| BR_COFINS_76 | COFINS 7.6% |

Os tax codes S/4 sao:

| MWSKZ | Descricao |
|-------|-----------|
| I1 | ICMS 18% |
| I2 | ICMS 12% |
| I3 | ICMS 7% |
| J1 | IPI 5% |
| J0 | IPI isento |
| P1 | PIS 1.65% |
| C1 | COFINS 7.6% |

### Tarefa 1: Preencha a TXS_ACTIVATION

| KAPPL | KALSM | ACTIVE |
|-------|-------|--------|
| | | |
| | | |

### Tarefa 2: Preencha a TXS_TAX_CODE_MAP (para MM)

| COUNTRY | COND_USAGE | APPLICATION | PRICING_NAME | MWSKZ | EXTERNAL_TAX_CODE |
|---------|-----------|-------------|-------------|-------|------------------|
| | | | | | |
| | | | | | |
| | | | | | |
| | | | | | |
| | | | | | |
| | | | | | |
| | | | | | |

### Tarefa 3: E para SD? O que muda?

Quais entradas adicionais sao necessarias na TXS_TAX_CODE_MAP para SD?

---

### Respostas

**Tarefa 1:**

| KAPPL | KALSM | ACTIVE |
|-------|-------|--------|
| TX | TXBRA1 | X |
| V | TXBRA1 | X |

**Tarefa 2 (MM):**

| COUNTRY | COND_USAGE | APPLICATION | PRICING_NAME | MWSKZ | EXTERNAL_TAX_CODE |
|---------|-----------|-------------|-------------|-------|------------------|
| BR | A | TX | TXBRA1 | I1 | BR_ICMS_18 |
| BR | A | TX | TXBRA1 | I2 | BR_ICMS_12 |
| BR | A | TX | TXBRA1 | I3 | BR_ICMS_7 |
| BR | A | TX | TXBRA1 | J1 | BR_IPI_5 |
| BR | A | TX | TXBRA1 | J0 | BR_IPI_0 |
| BR | A | TX | TXBRA1 | P1 | BR_PIS_165 |
| BR | A | TX | TXBRA1 | C1 | BR_COFINS_76 |

**Tarefa 3 (SD):**
Mesmas 7 entradas, mas com APPLICATION = **V** em vez de TX. Total: **14 entradas** (7 MM + 7 SD).
