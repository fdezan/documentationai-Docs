## 1. Top 5 Problemas do TXS e Como Resolver

### Problema 1: TXS Nao e Chamado — Condicoes de Imposto Zeradas

**Sintoma**: Conditions de imposto (TXSC, BX16, BIP0, BPI1, BCO1...) nao aparecem ou ficam zeradas na PO/SO.

**Fluxograma de Diagnostico:**

```
Condicoes de imposto zeradas?
    |
    v
Formula 279 atribuida no pricing procedure?
    |--- NAO → Atribuir Formula 279 nas condition types (TXSC + conditions de imposto)
    |--- SIM ↓
    v
TXS ativo na TXS_ACTIVATION para o cenario?
    |--- NAO → Criar entrada: KAPPL (TX ou V) + KALSM (procedure) + ACTIVE = X
    |--- SIM ↓
    v
Ha excecao na TXS_ACTIVATION_EXCEPTION?
    |--- SIM → Remover excecao ou ativar para a empresa/centro
    |--- NAO ↓
    v
Conexao HTTP esta ativa (Logical Port + OAuth2)?
    |--- NAO → Verificar Communication Arrangement
    |--- SIM → Verificar ADT: CL_TXS_SWITCH::IS_ACTIVE (variavel r_active)
```

**Classe-chave**: `CL_TXS_SWITCH` → metodo `IS_ACTIVE`
**Breakpoint ADT**: Inspecionar `r_active` (true/false) e a sequencia de verificacoes

### Problema 2: ICMS Retorna Zero ou Taxa Errada

**Sintoma**: Condition de ICMS (ex: BX16) preenchida, mas com KWERT = 0 ou KBETR incorreto.

**Fluxograma de Diagnostico:**

```
ICMS zerado ou taxa errada?
    |
    v
Sovos retornou tax_type ICMS na resposta?
    |--- NAO → Problema no Sovos (verificar configuracao do cenario no Sovos)
    |--- SIM ↓
    v
O valor (value) e taxa (rate) estao corretos na resposta?
    |--- NAO → Dados de entrada errados (NCM, UF, CNAE, contribuinte)
    |--- SIM ↓
    v
Mapeamento condition type → ICMS esta correto?
    |--- NAO → Verificar TXS_COND_MAP e J_1BNFTXCOND
    |--- SIM ↓
    v
Ha exempted_base_percent reduzindo a taxa?
    |--- SIM → Verificar se a reducao e esperada (base reduzida)
    |--- NAO → Verificar ADT: CL_TXS_CONDITIONS_MAPPER_BR::MAP_TAX_TYPE_TO_CONDITION
```

**Classe-chave**: `CL_TXS_CONDITIONS_MAPPER_BR` → metodo `MAP_TAX_TYPE_TO_CONDITION`
**Variaveis ADT**:
- `lt_available_tax_types` — tipos retornados pelo Sovos
- `ls_tax_type` — tipo sendo mapeado (rate, value, taxable)
- `c_pricing_condition_value` — KWERT resultante

### Problema 3: Erro de Tax Code ("tax_code_config_error")

**Sintoma**: Excecao `CX_TXS_TAX_CODE_CONFIG_ERROR` ao salvar documento.

**Fluxograma de Diagnostico:**

```
Erro CX_TXS_TAX_CODE_CONFIG_ERROR?
    |
    v
Tax code do Sovos existe na TXS_TAX_CODE_MAP?
    |--- NAO (0 matches) → Adicionar entrada na tabela
    |--- SIM ↓
    v
Ha multiplas entradas para o mesmo external_tax_code?
    |--- SIM (N matches) → Remover duplicatas ou especificar jurisdicao
    |--- NAO ↓
    v
Os campos-chave estao corretos?
    |  COUNTRY = BR?
    |  COND_USAGE = A?
    |  APPLICATION = TX (MM) ou V (SD)?
    |  PRICING_NAME = procedure correto?
    |--- NAO → Corrigir campos-chave
    |--- SIM → Verificar ADT: CL_TXS_TAX_CODE_DAO::GET_S4_TAX_CODE_SINGLE
```

**Classe-chave**: `CL_TXS_TAX_CODE_DAO` → metodo `GET_S4_TAX_CODE_SINGLE`
**Variaveis ADT**:
- `i_tax_code` — tax code externo recebido do Sovos
- `e_s4_tax_code` — tax code S/4 resultante
- Contar matches: 0 = nao configurado, N = ambiguidade

**Tabelas de Customizing** (prioridade):
1. `TXS_TAX_CODE_MAP_C` (customizada — prioridade)
2. `TXS_TAX_CODE_MAP` (standard)

### Problema 4: NCM Vazio ou Errado no Payload

**Sintoma**: Campo `item_classifications[].isc_code` (NCM) vazio ou mal formatado.

**Fluxograma de Diagnostico:**

```
NCM vazio ou errado no payload?
    |
    v
MARC-STEUC preenchido no centro do item?
    |--- NAO → Preencher STEUC no mestre de material (MM01 → Comercio Exterior)
    |--- SIM ↓
    v
Formato correto? (8 digitos, sem pontos)
    |--- NAO → STEUC com menos de 8 digitos → completar no mestre
    |--- SIM ↓
    v
E cenario STO?
    |--- SIM → Verificar STEUC no centro de ORIGEM tambem
    |--- NAO → Verificar ADT: CL_TXS_ITEM_CLASSIFICATIONS_BR::GET_MATERIAL_NCM
```

**Classe-chave**: `CL_TXS_ITEM_CLASSIFICATIONS_BR` → metodo `GET_MATERIAL_NCM`
**Formatacao**: `CL_TXS_MATERIAL_INFO_FORMATTER::FORMAT_NCM()` — remove pontos, pega 8 chars

---

### Problema 5: Timeout ou Erro de Conexao HTTP

**Sintoma**: Timeout, erro OAuth2, ou erro de rede na chamada ao Sovos.

**Fluxograma de Diagnostico:**

```
Erro de conexao HTTP?
    |
    v
Logical Port configurado e ativo?
    |--- NAO → Configurar Communication Arrangement
    |--- SIM ↓
    v
OAuth2 (Client Credentials) funcionando?
    |--- NAO → Verificar client_id, client_secret, token URL
    |--- SIM ↓
    v
Sovos esta respondendo? (testar URL diretamente)
    |--- NAO → Problema no Sovos (indisponibilidade, manutencao)
    |--- SIM ↓
    v
Firewall permite conexao de saida para o Sovos?
    |--- NAO → Solicitar liberacao de firewall (SAP BTP → Sovos)
    |--- SIM → Verificar ADT: CL_TXS_CONNECTION::GET_REST_CLIENT
```

**Classe-chave**: `CL_TXS_CONNECTION` → metodo `GET_REST_CLIENT`
**Sequencia de verificacao ADT**:
1. `conn_config` — configuracao Cloud vs On-Premise
2. `check_active_connection()` — conexao ativa?
3. `create_oauth2_client()` — Client Credentials Flow OK?
4. `create_http_client()` — HTTP client via Logical Port
5. `create_rest_client()` — REST client criado?

## 2. Ferramentas de Diagnostico

### Application Log (SLG1)

O TXS registra informacoes detalhadas no log de aplicacao:

| Informacao | Descricao | Quando Registra |
|-----------|-----------|-----------------|
| **Excecao** | Mensagem de erro completa | Sempre que ha erro |
| **Payload JSON** | JSON completo enviado ao Sovos | Sempre |
| **Response JSON** | JSON completo recebido do Sovos | Quando toggle ativo |
| **Correlation ID** | UUID do payload (X-CorrelationID) | Sempre |

**Para ativar log da resposta**: Ativar feature toggle `GLO_LOG_BR_EXT_JSON_RESP`

**Como usar**:
1. Reproduzir o erro (criar/alterar PO/SO)
2. Ir para SLG1 (ou app Fiori equivalente)
3. Filtrar por objeto/classe do TXS
4. Examinar o payload JSON (dados enviados)
5. Examinar o response JSON (dados recebidos)
6. Usar o Correlation ID para buscar no log do Sovos

### Breakpoints ADT Estrategicos

| Classe | Metodo | O Que Verificar | Fase |
|--------|--------|-----------------|------|
| `CL_TXS_SWITCH` | `IS_ACTIVE` | TXS esta ativo? (`r_active`) | 1 |
| `CL_TXS_PAYLOAD` | `BUILD_HEADER` | O que vai no header do request? | 3 |
| `CL_TXS_ITEMS_BR` | `BUILD` | O que vai nos itens? (NCM, tax_code, quantity) | 3 |
| `CL_TXS_LOCATIONS_BR` | `BUILD` | Localizacoes corretas? (UF, CEP) | 3 |
| `CL_TXS_PARTY_BR` | `BUILD` | CNPJ, IE, CNAE corretos? | 3 |
| `CL_TXS_COMMUNICATION` | `POST` | Payload enviado e resposta recebida | 4 |
| `CL_TXS_RESPONSE_BR` | Constructor | Resposta desserializada | 5 |
| `CL_TXS_CONDITIONS_MAPPER_BR` | `MAP_TAX_TYPE_TO_CONDITION` | Como mapeia tax_type → KOMV? | 6 |
| `CL_TXS_TAX_CODE_DAO` | `GET_S4_TAX_CODE_SINGLE` | Tax code encontrado? | 6 |
| `CL_TXS_CONDITION_PROCESSING_BR` | `CHECK_MAPPING` | Pelo menos 1 condicao mapeada? | 7 |

### Headers HTTP para Rastreamento

| Header | Valor | Uso |
|--------|-------|-----|
| `X-CorrelationID` | UUID do payload | Rastrear no log do Sovos |
| `X-Routing-Key` | Chave do parceiro | Identificar roteamento |
| `Content-Type` | application/json | Sempre JSON |
| `Connection` | keep-alive | Reutilizar conexao |

### Response Buffer

O `CL_TXS_RESPONSE_BUFFER` armazena a resposta do Sovos em memoria:

- **Chave**: hash do payload + contexto de pricing
- **Proposito**: Evitar chamadas HTTP duplicadas (1 call, N mapeamentos)
- **Erro comum**: Se TXSC falha, as demais conditions (BX16, BIP0, BPI1...) nao encontram resposta no buffer → `CX_TXS_RESPONSE_NOT_ON_BUFFER`
- **Solucao**: Verificar se a condition TXSC (service call) executou com sucesso antes

## 3. Extensibilidade — Pontos de Extensao

### BAdI Released: BD_GSLOG_EXT_TAX_CALC_PAYLOAD (Seguro para Cloud)

| Aspecto | Detalhe |
|---------|---------|
| **Status** | Released (C1) — seguro para Public Cloud |
| **Fase** | 3 (Build Payload) — depois da montagem, antes do HTTP |
| **Interface** | IF_GSLOG_EXT_TAX_CALC_PAYLOAD |
| **Metodo** | `extend_payload()` |
| **Risco** | Baixo — compativel com upgrades automaticos |

**Assinatura:**
```
methods EXTEND_PAYLOAD
  importing
    I_PRICING_HEADER   type KOMK          " Header do pricing
    I_PRICING_ITEM     type KOMP          " Item do pricing
    I_LFD_FOR_CONSUMPTION type J_1BLFD    " Dados fiscais (LFD)
  changing
    C_PAYLOAD          type TXS_PAYLOAD.  " Payload completo — pode modificar
```

**O que voce pode modificar:**
- `item_classifications[]` — adicionar/alterar NCM, CEST customizados
- `additional_item_information[]` — adicionar CFOP custom, flags
- `party[].tax_registration[]` — ajustar CNPJ, IE, CNAE
- `bp_exemption_details[]` — detalhes de isencao
- `locations[]` — ajustar enderecos
- Campos do header

**Caso de uso tipico**: Enviar campo adicional ao Sovos que o standard nao envia (ex: tipo de operacao especifica do cliente).

### Custom Fields V2 — YY1_* (Key User Extensibility)

| Aspecto | Detalhe |
|---------|---------|
| **Disponibilidade** | Payload V2 apenas (SD) |
| **Prerequisito** | Integracao V2 ativa |
| **Risco** | Baixo — sem codigo, sem BAdI |

**Como funciona:**
1. Criar campo customizado via app Fiori "Custom Fields" (F1481)
2. Campo recebe prefixo automatico `YY1_`
3. Na proxima chamada TXS V2, o campo aparece em `extensions[]` ou `item_extensions[]`
4. Sovos recebe e pode usar na determinacao fiscal

**Exemplo no payload:**
```json
{
  "extensions": [
    { "customField": "YY1_SPECIAL_TAX_FLAG", "customFieldValue": "X" }
  ],
  "items": [{
    "itemExtensions": [
      { "customField": "YY1_MATERIAL_GROUP_2", "customFieldValue": "MINING" }
    ]
  }]
}
```

**Classe interna**: `CL_TXS_CUSTOM_FIELD_EXTRACTOR` — usa RTTI para inspecionar KOMK e extrair campos YY1_*

### Tabelas de Customizing (Extensao por Configuracao)

| Tabela | Proposito | Risco |
|--------|----------|-------|
| `TXS_TAX_CODE_MAP_C` | Mapeamento customizado de tax codes | Baixo |
| `TXS_COND_MAP_C` | Mapeamento customizado de condition types + classe custom | Medio |

### BAdI NAO Released: TXS_BADI_CONTROLLER (Alerta!)

| Aspecto | Detalhe |
|---------|---------|
| **Status** | NAO Released — pode quebrar em upgrades |
| **Interface** | IF_EX_TXS_BADI_CONTROLLER |
| **Risco** | Alto |

**Metodos disponiveis (NAO recomendados para clientes):**

| Metodo | Fase | O Que Faz |
|--------|------|-----------|
| `pre_processing()` | 1 | Intercepta antes do TXS (pode desabilitar TXS completamente) |
| `post_processing()` | 8 | Ajusta valores apos mapeamento |
| `error_handling()` | 8 | Tratamento customizado de erros |
| `calculate_taxes()` | 4 | Substitui chamada HTTP (usado por Sovos/Vertex internamente) |

**Quem usa**: Parceiros como Sovos e Vertex implementam `calculate_taxes()` para integrar seu proprio motor.
**Para clientes**: Use apenas o BAdI Released (BD_GSLOG_EXT_TAX_CALC_PAYLOAD) ou Custom Fields V2.

### Matriz de Decisao — Qual Extensao Usar?

| Necessidade | Extensao | Risco | Recomendado? |
|------------|----------|-------|-------------|
| Adicionar campo ao payload antes do HTTP | BAdI BD_GSLOG_EXT_TAX_CALC_PAYLOAD | Baixo | SIM |
| Enviar campos customizados automaticamente | Custom Fields YY1_* + Payload V2 | Baixo | SIM |
| Mapear tax code customizado | TXS_TAX_CODE_MAP_C | Baixo | SIM |
| Usar classe de mapeamento propria | TXS_COND_MAP_C + classe custom | Medio | Com cautela |
| Interceptar antes/depois do TXS | BAdI TXS_BADI_CONTROLLER | Alto | NAO (exceto parceiros) |
| Usar motor de calculo proprio | BAdI TXS_BADI_CONTROLLER::calculate_taxes | Alto | NAO (exceto parceiros) |

### Cenario C: Erro "tax_code_config_error"

**Situacao**: Ao salvar uma Sales Order, aparece o erro `CX_TXS_TAX_CODE_CONFIG_ERROR`. O log mostra que o Sovos retornou o tax code externo `BR_ICMS_ST_18`.

**Informacoes adicionais**:

### Respostas dos Cenarios

**Cenario A**: Provavel excecao na `TXS_ACTIVATION_EXCEPTION` para o centro BR01. Como outros centros funcionam (BR02), a ativacao geral esta OK, mas ha uma excecao especifica desativando o TXS para BR01. **Solucao**: Verificar e remover a excecao na TXS_ACTIVATION_EXCEPTION.

**Cenario B**: O problema esta nos dados de entrada enviados ao Sovos, nao na configuracao do TXS. Possiveis causas: (1) endereco do cliente com UF errada (MG em vez de SP); (2) endereco do BP Ship-To diferente do Sold-To; (3) CNAE indicando operacao interestadual. **Solucao**: Verificar o payload JSON no log (SLG1) e confirmar que as UFs de origem e destino estao corretas.

**Cenario C**: O Sovos retornou um tax code externo (`BR_ICMS_ST_18`) que nao existe na TXS_TAX_CODE_MAP. E um cenario de ICMS-ST que nao foi configurado. **Solucao**: Adicionar entrada na TXS_TAX_CODE_MAP para BR_ICMS_ST_18 mapeando para o tax code S/4 de ICMS-ST correspondente.
