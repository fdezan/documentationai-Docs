O **Tax Service (TXS)**, tambem conhecido como TAAS (Tax As A Service) ou Tax Engine, e o metodo de calculo de impostos brasileiros para o **S/4HANA Cloud Public**.

Diferente do metodo classico (CBT/TAXBRA) que calcula impostos internamente via Condition Records, o TXS envia os dados fiscais para um **motor externo** (ex: Sovos, Thomson Reuters) via chamada HTTP REST e recebe de volta os valores de impostos calculados.

---

## Arquitetura — 8 Fases

O pipeline TXS executa em **8 fases** sequenciais:

| Fase | Nome | Classe Principal | O que faz |
|------|------|-----------------|-----------|
| 1 | Entrada | `CL_TXS_CNDNS_HANDLE_BR` | Ponto de entrada via formula 279 no pricing |
| 2 | Validacao | `CL_TXS_SWITCH` | Verifica se TXS esta ativo para a combinacao app + procedure + pais |
| 3 | Build Payload | `CL_TXS_PAYLOAD_BR` | Constroi o JSON com header, items, locations, parties, extensions |
| 4 | Comunicacao | `CL_TXS_COMMUNICATION` | Serializa JSON, OAuth2, chamada HTTP REST, parse da response |
| 5 | Response Parse | `CL_TXS_RESPONSE_BR` | Processa tax types (ICMS, IPI, PIS, COFINS, etc.) e tax values |
| 6 | Condition Mapping | `CL_TXS_CONDITIONS_MAPPER_BR` | Mapeia tax values para condition types no pricing (KOMV) |
| 7 | Verificacao | `CL_TXS_CNDNS_EXCEPTIONS_HANDLE_BR` | Valida mapeamento, error handling, SLG1 logging |
| 8 | Pos-Processamento | `GSLOG_LOCAL_FISCAL_DATA` | Grava dados fiscais locais (LFD), atualiza NF |
