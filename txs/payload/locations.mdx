## Visao Geral

A estrutura `TXS_LOCATION` representa os enderecos envolvidos em uma transacao fiscal enviada ao Tax Service (TXS). Cada location possui um **tipo** (ship_from, ship_to, contract_from, contract_to, service_location) que indica a funcao do endereco na operacao.

O Brasil (BR) utiliza multiplos cenarios de locations dependendo da aplicacao e do tipo de operacao:

- **MM Material** â€” compra de material (fornecedor -> planta)
- **MM Servico** â€” compra de servico (contract_from, contract_to, service_location)
- **MM Frete** â€” frete incoming (transportador, fornecedor da mercadoria, endereco de entrega)
- **MM Consignment Return** â€” retorno de consignacao (inversao planta <-> fornecedor)
- **SD Material** â€” venda de material (planta -> cliente)
- **SD Servico** â€” venda de servico (contract_from, contract_to, service_location)
- **STO** â€” transferencia entre plantas (planta fornecedora -> planta receptora)

A classe base `CL_TXS_LOCATIONS` implementa a logica generica, e `CL_TXS_LOCATIONS_BR` herda e redefine metodos para o cenario brasileiro. O pattern e **Country Specialization**: a factory (`CL_TXS_FACTORY`) cria automaticamente a instancia `_BR` quando o pais e Brasil.

**Fontes**:
- Estrutura DDIC: `txs_location.abap`
- Classe base: `cl_txs_locations.abap`
- Especializacao BR: `cl_txs_locations_br.abap`

## Estrutura: TXS_LOCATION

Definida como DDIC Structure com annotation `@AbapCatalog.enhancement.category : #EXTENSIBLE_ANY`.
(Fonte: txs_location.abap:1-30)

| Campo | Tipo ABAP | Label | Descricao Funcional |
|-------|-----------|-------|---------------------|
| `type` | `abap.string(0)` | The type of address | Tipo de endereco: valor resolvido via internal codes (SHIP_FROM, SHIP_TO, CONTRACT_FROM, CONTRACT_TO, SERVICE_LOCATION). O valor real enviado na API depende da configuracao em `CL_TXS_INTERNAL_CODES`. |
| `address_line1` | `abap.string(0)` | First address line | Logradouro principal. Fonte tipica: `LFA1-STRAS`, `T001W-STRAS` (via `get_plants_or_branches`), `KNA1-STRAS` (via `get_customer_data`). |
| `address_line2` | `abap.string(0)` | Second address line | Complemento de endereco (pouco usado no Brasil). |
| `address_line3` | `abap.string(0)` | Third address line | Terceira linha de endereco (pouco usado no Brasil). |
| `zip_code` | `abap.string(0)` | Zip or Postal code | CEP. Fonte tipica: `LFA1-PSTLZ`, `T001W-PSTLZ`, `KNA1-PSTLZ`. |
| `city` | `abap.string(0)` | City | Cidade. Fonte tipica: `LFA1-ORT01`, `T001W-ORT01`, `KNA1-ORT01`. |
| `county` | `abap.string(0)` | County | Municipio/County. Usado em MM para planta (`get_plants_or_branches-counc`), SD ship_from (`get_plants_or_branches-counc`), e frete ship_to (`ls_address_data-county`). Em SD ship_to, e limpo (`SPACE`). |
| `state` | `abap.string(0)` | State or province code | UF/Estado. Fonte tipica: `LFA1-REGIO`, `T001W-REGIO`, `KNA1-REGIO`. |
| `country` | `abap.string(0)` | 2 letter ISO 3166-1 alpha-2 country code | Pais ISO (ex: `BR`). Fonte tipica: `LFA1-LAND1`, `T001W-LAND1`, `KNA1-LAND1`. |
| `address_id` | `abap.string(0)` | Address ID as in the Tax Configuration application | ID do endereco conforme configurado no app Tax Configuration. Nao preenchido diretamente no builder de locations. |
| `is_company_tax_registered` | `abap.string(0)` | Is the company registered with the tax authority | Indica se a empresa propria esta registrada na autoridade fiscal. **BR redefine** para corpo vazio (limpa o campo). Na classe base, verifica VAT Number (`STCEG`) via `DAO->get_domestic_vat_reg_number`. |
| `is_bp_tax_registered` | `abap.string(0)` | Business to Business (B2B) or Business to Consumer (B2C) | Indica B2B vs B2C. **BR redefine** para corpo vazio (limpa o campo). Na classe base, verifica `STCEG` do fornecedor/cliente. |

**Observacao sobre os campos `is_company_tax_registered` e `is_bp_tax_registered` no Brasil**: A classe `CL_TXS_LOCATIONS_BR` redefine ambos os metodos com corpo vazio (linhas 381-388 de `cl_txs_locations_br.abap`). Alem disso, nos metodos `GET_LOCATION_MM_SHIP_FROM`, `GET_LOCATION_MM_SHIP_TO`, `GET_LOCATION_SD_SHIP_FROM` e `GET_LOCATION_SD_SHIP_TO`, apos obter o location, esses campos sao explicitamente limpos com `CLEAR` (ex: linhas 341-342, 356-357, 366-367, 376-377). Isso indica que o Brasil **nao usa** esses campos nas locations para determinacao fiscal â€” a logica de contribuinte e tratada por outros mecanismos (Party/Tax Registration).

## Dispatch Principal (BR)

O metodo `ADD_LOCATIONS` e redefinido em `CL_TXS_LOCATIONS_BR` para adicionar suporte a STO antes do dispatch padrao MM/SD:

```abap
method add_locations.
  if pricing_context->is_stock_transfer( ).
    add_locations_sto( ).          " STO: transferencia entre plantas
  else.
    case pricing_context->get_application_for_payload( ).
      when if_txs_constants=>application-taxes.           " 'TX' (MM)
        add_locations_mm( ).
      when if_txs_constants=>application-sales_and_distribution.  " 'V' (SD)
        add_locations_sd( ).
    endcase.
  endif.
endmethod.
```

(Fonte: cl_txs_locations_br.abap:61-72)

**Valores das aplicacoes**:
- `if_txs_constants=>application-taxes` = `'TX'` (MM â€” Materials Management)
- `if_txs_constants=>application-sales_and_distribution` = `'V'` (SD â€” Sales & Distribution)

(Fonte: if_txs_constants.abap:7-8)

### Comportamento especial BR â€” Transferencia para Subcontratacao

A redefinicao BR em `GET_LOCATION_MM_SHIP_FROM` e `GET_LOCATION_MM_SHIP_TO` verifica se a operacao e uma transferencia para subcontratado (`IS_TRNSFR_PSTNG_TO_SUBCON`). Se for, **inverte** a origem dos dados:

```abap

if is_trnsfr_pstng_to_subcon( ) = abap_false.
  r_location = get_location_mm_mat_supplier( ... ).  " Normal: fornecedor
else.
  r_location = get_location_mm_mat_plant( ... ).     " Subcon: planta
endif.
clear r_location-is_bp_tax_registered.
clear r_location-is_company_tax_registered.

" GET_LOCATION_MM_SHIP_TO (BR):
if is_trnsfr_pstng_to_subcon( ) = abap_false.
  r_location = get_location_mm_mat_plant( ... ).     " Normal: planta
else.
  r_location = get_location_mm_mat_supplier( ... ).  " Subcon: fornecedor
endif.
clear r_location-is_bp_tax_registered.
clear r_location-is_company_tax_registered.
```

(Fonte: cl_txs_locations_br.abap:330-358)

A deteccao de subcontratacao usa `CL_TXS_MAT_DOC_CONTEXT` para verificar o movement type do documento de material:

```abap

  data(lr_mat_doc_context) = cl_txs_mat_doc_context=>get_instance( ).
  data(ls_item_to_process) = lr_mat_doc_context->get_mat_doc_item_to_process(
    pricing_context->get_item( ) ).
```

### SHIP_TO â€” Endereco de Entrega (Frete)

O SHIP_TO de frete usa o endereco de entrega do PO de referencia via `CL_TXS_MM_ADDRESS_ADAPTER`. Se nao houver delivery address, mantem o endereco da planta (que vem do `GET_LOCATION_MM_SHIP_TO` chamado primeiro).

```abap
method add_locations_mm_frght_ship_to.
  data(ls_location_frght_ship_to) = get_location_mm_ship_to( ).  " Base: planta
  " Obtem delivery address do PO de referencia
  data(lv_ebelp) = pricing_context->get_item( )-evrtp.
  data(lv_ebeln) = pricing_context->get_item( )-evrtn.
  data(ls_address_data) = cl_txs_mm_address_adapter=>get_instance(
    i_ebelp = lv_ebelp
    i_ebeln = lv_ebeln )->get_delivery_data( ).
  " Se houver delivery address, sobrescreve os campos
  if ls_address_data is not initial.
    ls_location_frght_ship_to-address_line1 = ls_address_data-street.
    ls_location_frght_ship_to-zip_code = ls_address_data-post_code1.
    ls_location_frght_ship_to-county = ls_address_data-county.
    ls_location_frght_ship_to-city = ls_address_data-city1.
    ls_location_frght_ship_to-state = ls_address_data-region.
    ls_location_frght_ship_to-country = ls_address_data-country.
  endif.
  append_location( ls_location_frght_ship_to ).
endmethod.
```

### IS_BP_TAX_REGISTERED (redefinido)

O Brasil redefine `IS_BP_TAX_REGISTERED` com **corpo vazio** â€” nao retorna nenhum valor.

(Fonte: cl_txs_locations.abap:520-559)

O Brasil tambem redefine `IS_COMPANY_TAX_REGISTERED` com **corpo vazio**.

```abap
method is_company_tax_registered.
  " Corpo vazio â€” nao determina contribuinte via location no Brasil
endmethod.
```

(Fonte: cl_txs_locations_br.abap:386-388)

Na **classe base**, esse metodo verifica o VAT Registration Number da empresa:

(Fonte: cl_txs_locations.abap:562-615)
