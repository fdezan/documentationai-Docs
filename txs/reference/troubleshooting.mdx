Diagnostico de problemas comuns do Tax Service com breakpoints ADT e checklist de verificacao.

**Uso**: Para diagnostico guiado, use a skill `/diagnosticar-txs`.

---

## Cenarios de Problema

### 1. Pricing nao calcula impostos (TXS nao e chamado)

**Sintoma**: Conditions de imposto TXS (TXSC e demais como ICM1, BIP0, IPIS, ICOF, etc.) nao aparecem ou ficam zeradas no pricing.

**Checklist**:
1. Formula 279 esta atribuida as condition types na pricing procedure?
2. TXS esta ativo para a combinacao application + pricing procedure + pais?
3. Existe excecao de ativacao para a empresa/planta?

**Classe**: CL_TXS_SWITCH
**Breakpoint ADT**: Metodo `IS_ACTIVE`
**Variaveis a inspecionar**:
- `r_active` — resultado final (true/false)
- Verificar sequencia: conexao ativa → ativacao por app+procedure → excecoes → V2 control

**Feature Toggles**:
- Tabela TXS_ACTIVATION: ativacao por application (V/TX) + pricing procedure (TXBRA1, etc.)
- Tabela TXS_ACTIVATION_EXCEPTION: excecoes por empresa, planta

---

### 2. ICMS retorna zero / aliquota incorreta

**Sintoma**: Conditions de ICMS (BX16, ICM1, ICM2, etc.) preenchidas mas com valor 0 ou aliquota errada.

**Checklist**:
1. O Tax Engine retornou tax_type ICMS na response? (verificar JSON)
2. O mapeamento na J_1BNFTXCOND esta correto para as conditions ICMS?
3. A base de calculo (taxable) esta correta?
4. Existe exempted_base_percent reduzindo a aliquota?

**Classe**: CL_TXS_CONDITIONS_MAPPER_BR
**Breakpoint ADT**: Metodo `MAP_TAX_TYPE_TO_CONDITION`
**Variaveis a inspecionar**:
- `lt_available_tax_types` — tax types retornados pelo engine
- `ls_tax_type` — o tax type sendo mapeado (rate, value, taxable)
- `c_pricing_condition_value` — KWERT resultante

**Segundo breakpoint**: CL_TXS_TAX_TYPES_VALUES_BR, metodo `BUILD_MM_ICMS` ou `BUILD_SD_ICMS`
- Verificar se `tax_value` do ICMS tem rate e value preenchidos

---

### 3. Erro "mapping_condition_error" — Nenhuma condition mapeou

**Sintoma**: Mensagem de erro indicando que nenhuma condition type foi mapeada com sucesso.

**Checklist**:
1. TODAS as conditions retornaram KINAK = 'X' (exclusion)?
2. O Tax Engine retornou algum tax_type na response?
3. Os condition types na pricing procedure correspondem aos tax types retornados?

**Classe**: CL_TXS_CONDITION_PROCESSING / CL_TXS_CONDITION_PROCESSING_BR
**Breakpoint ADT**: Metodo `CHECK_MAPPING`
**Logica**:
- Se e a ULTIMA condition (formula 279) e NENHUMA mapeou com sucesso → ERRO
- Se pelo menos 1 condition ja mapeou → OK

**Solucao tipica**: Verificar se a pricing procedure tem os condition types corretos — TXSC (service call) + conditions de imposto configuradas na J_1BTXCOND (ex: BX16/ICM1 para ICMS, BIP0/IPI1 para IPI, IPIS/BPI1 para PIS, ICOF/BCO1 para COFINS).

---

### 4. Tax code invalido / erro de configuracao

**Sintoma**: `CX_TXS_TAX_CODE_CONFIG_ERROR` — tax code do engine nao encontrado no S/4.

**Checklist**:
1. O tax code retornado pelo engine existe na tabela TXS_TAX_CODE_MAP?
2. O mapeamento usa o pais, application e pricing procedure corretos?
3. Existe ambiguidade (N matches para o mesmo tax code externo)?

**Classe**: CL_TXS_TAX_CODE_DAO
**Breakpoint ADT**: Metodo `GET_S4_TAX_CODE_SINGLE`
**Variaveis a inspecionar**:
- `i_tax_code` — tax code externo recebido do engine
- `e_s4_tax_code` — tax code S/4 resultante
- Se 0 matches: tax code nao configurado
- Se N matches: ambiguidade na tabela

**Tabelas de customizing**:
- TXS_TAX_CODE_MAP_C (customizing, prioridade)
- TXS_TAX_CODE_MAP (standard)
- Chave: COUNTRY + COND_USAGE + APPLICATION + PRICING_NAME + EXTERNAL_TAX_CODE

### 5. HTTP timeout / erro de conexao

**Sintoma**: Timeout na chamada HTTP ao Tax Engine, erro OAuth2, erro de rede.

**Checklist**:
1. Logical port esta configurado e ativo?
2. Credenciais OAuth2 (Client Credentials) estao corretas?
3. O Tax Engine esta respondendo? (testar URL diretamente)
4. Existe firewall bloqueando a saida?

**Classe**: CL_TXS_CONNECTION
**Breakpoint ADT**: Metodo `GET_REST_CLIENT`
**Sequencia de verificacao**:
1. `conn_config` — configuracao Cloud vs On-Premise
2. `check_active_connection()` — conexao ativa?
3. `create_oauth2_client()` — Client Credentials Flow funciona?
4. `create_http_client()` — HTTP client via Logical Port
5. `create_rest_client()` — REST client criado?
6. `set_token()` — token OAuth2 atribuido?

**Segundo breakpoint**: CL_TXS_COMMUNICATION, metodo `POST`
- Verificar HTTP status code na response
- Se status != 200 → `deserialize_error()` → `cx_txs_exception(http_server_error)`

---

### 6. NCM incorreto ou vazio no payload

**Sintoma**: Campo item_classifications[].isc_code (NCM) vazio ou com formato errado.

**Checklist**:
1. KOMP-STEUC esta preenchido? (NCM vem do cadastro de material MARC-STEUC)
2. O formato esta correto? (8 digitos, sem pontos — ex: 84719012)
3. Para STO: a planta de origem tem STEUC preenchido?

**Classe**: CL_TXS_ITEM_CLASSIFICATIONS_BR
**Breakpoint ADT**: Metodo `GET_MATERIAL_NCM`
**Variaveis a inspecionar**:
- `pricing_context->get_item()->steuc` — valor bruto do NCM
- Apos `format_ncm()`: resultado formatado (8 chars, sem pontos)

**Segundo breakpoint**: CL_TXS_MATERIAL_INFO_FORMATTER, metodo `FORMAT_NCM`
- Remove pontos, toma 8 caracteres

---

### 7. CNAE vazio no payload

**Sintoma**: Campo party[].tax_registration[].number (CNAE) vazio.

**Checklist** (~~BRF+ descontinuado no TXS~~ — CNAE agora vem dos dados mestre):
1. O CNAE esta preenchido nos dados mestre do parceiro? (`LFA1-CNAE` / `KNA1-CNAE`)
2. O CNAE esta preenchido na planta? (`J_1BBRANCH-CNAE`)
3. O cenario e STO? Se sim, CNAE vem de `J_1BBRANCH-CNAE` de ambas as plantas.

**Classe**: `CL_TXS_PARTY_BR`
**Breakpoint ADT**: `GET_VALUE_CNAE` (linha 1867)

**Solucao**: Preencher CNAE diretamente nos dados mestre: `LFA1-CNAE`, `KNA1-CNAE` ou `J_1BBRANCH-CNAE`.

---

### 8. Payload V1 quando deveria ser V2 (ou vice-versa)

**Sintoma**: Payload enviado no formato errado — faltam campos V2 (parties no item, extended fields).

**Checklist**:
1. Qual application? V = SD, TX = MM
2. Para V2: integracao ativa (Activation Control V2)?
3. Para V1 com auto tax code: MWSKZ preenchido?

**Classe**: CL_TXS_PAYLOAD_TYPE
**Breakpoint ADT**: Metodo `DETERMINE_PAYLOAD_TYPE`
**Logica**:
- SD + integracao V2 ativa → V2 (CL_TXS_REQUEST_V2)
- MM + auto tax code → V1 (CL_TXS_PAYLOAD)
- MM default → DirectCalc (CL_TXS_DIRECT_CALC_PAYLOAD)

---

### 9. Response nao encontrada no buffer (2a+ condition)

**Sintoma**: `CX_TXS_RESPONSE_NOT_ON_BUFFER` — uma condition de imposto (ex: ICM1) nao encontra a response.

**Checklist**:
1. A condition de service call (TXSC) executou com sucesso?
2. A response foi adicionada ao buffer apos a chamada HTTP?
3. O payload + context usados como chave do buffer sao os mesmos?

**Classe**: CL_TXS_RESPONSE_BUFFER
**Breakpoint ADT**: Metodo `GET_RESPONSE`
**Variaveis a inspecionar**:
- Chave do buffer (payload + pricing_context)
- Se o buffer esta vazio → a chamada HTTP falhou na 1a condition

---

### 10. Impostos calculados mas KOMV nao preenchido

**Sintoma**: Tax Engine retorna valores corretos, mas as conditions no pricing ficam zeradas.

**Checklist**:
1. O mapper esta sendo instanciado corretamente? (CL_TXS_COND_MAPPER_FACTORY)
2. Existe classe customizada na TXS_COND_MAP? (pode estar interceptando)
3. Os tax_type_codes retornados correspondem aos esperados pelo mapper?

**Classe**: CL_TXS_COND_MAPPER_FACTORY
**Breakpoint ADT**: Metodo `INITIALIZE`
**Logica de instanciacao**:
1. Classe customizada configurada na TXS_COND_MAP? → CREATE OBJECT classe_customizada
2. V2 → CL_TXS_STANDARD_TAX_MAP_V2
3. V1 → CL_TXS_CONDITIONS_MAPPER_BR (ou fallback CL_TXS_STANDARD_TAX_MAP)

## Ferramentas de Diagnostico

### Application Log (SLG1)

**Classe**: CL_TXS_LOG_DAO
**Metodo**: `add_exception_to_log(exception, payload_json, response_json)`

O TXS grava logs com:
- Excecao completa (mensagem, classe, metodo)
- Payload JSON enviado
- Response JSON recebido
- Correlation ID (X-CorrelationID do header HTTP)

### Feature Toggles Relevantes

| Toggle | Descricao | Efeito |
|--------|-----------|--------|
| GLO_LOG_BR_EXT_JSON_RESP | Log de JSON response | Habilita log detalhado da response |
| TXS_ACTIVATION | Ativacao por app + procedure | Liga/desliga TXS por cenario |
| TXS_ACTIVATION_EXCEPTION | Excecoes de ativacao | Desliga TXS para empresa/planta especifica |

### Headers HTTP para Debug

| Header | Valor | Uso |
|--------|-------|-----|
| X-CorrelationID | payload.id (UUID) | Rastrear requisicao no Tax Engine |
| X-Routing-Key | partner_routing_key | Identificar parceiro de roteamento |
| Content-Type | application/json | Sempre JSON |
| Connection | keep-alive | Reutiliza conexao |
