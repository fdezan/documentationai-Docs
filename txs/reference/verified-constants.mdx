## 16. Deprecacoes e Mudancas SAP

### Uso de BRF+ Descontinuado no TXS

**Status**: USO NO TXS DESCONTINUADO pela SAP

A SAP descontinuou o uso de **BRF+** (Business Rule Framework Plus) **dentro do Tax Service (TXS)**. O BRF+ como tecnologia SAP continua funcionando normalmente para outros fins — o que foi removido e especificamente a integracao do TXS com BRF+ para determinacao de CNAE, SUFRAMA e ICMS-ST Contributor.

**Areas afetadas no codigo-fonte existente** (referencias ao BRF+ no TXS agora obsoletas):

| Funcionalidade | Classes que usavam BRF+ no TXS | Situacao Atual |
|----------------|-------------------------------|----------------|
| Determinacao de CNAE | `CL_TXS_CNAE_BR` (8 metodos: `CNAE_FOR_MM_BP_MAT`, `CNAE_FOR_MM_PLANT_MAT`, `CNAE_FOR_SD_BP_MAT`, etc.) | CNAE agora vem direto dos dados mestre (`LFA1-CNAE`, `KNA1-CNAE`, `J_1BBRANCH-CNAE`) |
| Determinacao de SUFRAMA | `CL_TXS_DAO_SUFRAMA_MM_BR`, `CL_TXS_SUFRAMA_DAO_BR` | SUFRAMA agora vem direto dos dados mestre (`J_1BBRANCH-SUFRAMA`, `KNA1-SUFRAMA`) |
| ICMS-ST Contributor | `CL_TXS_BRF_ICMSST_BR` | Contribuinte ICMS-ST via dados mestre / `BUS_TB071` |

**UUIDs BRF+ no TXS** (nao mais utilizados pelo TXS):

Os 8 BRF+ Function UUIDs para CNAE (ex: `40F2E9AFBE791ED9BEE645A291E3B35F`) e os UUIDs para SUFRAMA definidos em `if_txs_constants_br.abap:167-170` nao sao mais utilizados pelo TXS.

**Impacto para consultores**:
- Nao e mais necessario configurar regras BRF+ para CNAE/SUFRAMA no contexto do TXS
- CNAE e SUFRAMA devem ser mantidos diretamente nos dados mestre (LFA1, KNA1, J_1BBRANCH)
- Troubleshooting de CNAE/SUFRAMA vazio no TXS: verificar apenas os dados mestre, nao mais regras BRF+
- O BRF+ continua disponivel como tecnologia SAP para outros usos fora do TXS
