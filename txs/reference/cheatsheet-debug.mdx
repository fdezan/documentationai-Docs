## Cenarios de Troubleshooting

| # | Sintoma | Classe | Metodo | Variaveis a Inspecionar | Tabela Config | Solucao Tipica |
|:-:|---------|--------|--------|------------------------|:-------------:|----------------|
| **1** | TXS nao chamado — conditions TXS ausentes ou zeradas | `CL_TXS_SWITCH` | `IS_ACTIVE` | `r_active`, `me->connection_active`, `lv_integration_active`, `lv_exception_active` | `TXS_ACTIVATION`, `TXS_ACTIVATION_EXCEPTION` | Criar entrada em TXS_ACTIVATION para APPLICATION + PRICING_PROCEDURE + COUNTRY |
| **2** | ICMS retorna zero ou aliquota incorreta | `CL_TXS_CONDITIONS_MAPPER_BR` | `MAP_TAX_TYPE_TO_CONDITION` | `lt_available_tax_types`, `ls_tax_type` (rate, value, taxable), `c_pricing_condition_value` | `TXS_COND_MAP` | Corrigir UF nos dados mestres (LFA1-REGIO / T001W-REGIO) ou regra no Engine |
| **3** | Erro "mapping_condition_error" — nenhuma condition mapeou | `CL_TXS_CONDITION_PROCESSING` | `CHECK_MAPPING` | Lista de conditions mapeadas, flag de ultima condition | — | Verificar pricing procedure — TXSC (service call) + conditions de imposto configuradas na J_1BTXCOND (BX16/ICM1, BIP0/IPI1, IPIS, ICOF, etc.) |
| **4** | Tax code invalido (`CX_TXS_TAX_CODE_CONFIG_ERROR`) | `CL_TXS_TAX_CODE_DAO` | `GET_S4_TAX_CODE_SINGLE` | `i_tax_code` (externo), `e_s4_tax_code` (S/4) | `TXS_TAX_CODE_MAP`, `TXS_TAX_CODE_MAP_C` | Criar entrada na TXS_TAX_CODE_MAP para o tax code retornado pelo Engine |
| **5** | HTTP timeout ou erro OAuth2 / conexao | `CL_TXS_CONNECTION` | `GET_REST_CLIENT` | `conn_config`, OAuth2 client, HTTP client, REST client, token | — | Verificar Communication Arrangement (URL, OAuth2 credentials, rede) |
| **6** | NCM vazio ou formato incorreto no payload | `CL_TXS_ITEM_CLASSIFICATIONS_BR` | `GET_MATERIAL_NCM` | `pricing_context->get_item()->steuc`, resultado apos `format_ncm()` | — | Preencher MARC-STEUC no mestre de material para a planta correta |
| **7** | CNAE vazio no payload (party registration) | `CL_TXS_PARTY_BR` | `GET_VALUE_CNAE` | CNAE dos dados mestre: LFA1-CNAE / KNA1-CNAE / J_1BBRANCH-CNAE (~~BRF+ descontinuado no TXS~~) | — | Preencher CNAE nos dados mestre do parceiro (LFA1/KNA1-CNAE) ou planta (J_1BBRANCH-CNAE) |
| **8** | Payload V1 quando deveria ser V2 (ou vice-versa) | `CL_TXS_PAYLOAD_TYPE` | `DETERMINE_PAYLOAD_TYPE` | Aplicacao (MM/SD), feature toggle V2, tipo de payload retornado | — | Verificar feature toggles V2 (GLO_GEN_TXS_V2_INTEGRATION) |
| **9** | Buffer vazio na 2a condition (`CX_TXS_RESPONSE_NOT_ON_BUFFER`) | `CL_TXS_RESPONSE_BUFFER` | `GET_RESPONSE` | Buffer key (payload + pricing_context), conteudo do buffer | — | TXSC deve ser 1a condition TXS na procedure; resolver falha HTTP da TXSC |
| **10** | Impostos calculados mas KOMV zerada | `CL_TXS_COND_MAPPER_FACTORY` | `INITIALIZE` | Mapper instanciado (custom / V2 / V1-BR), classe do mapper | `TXS_COND_MAP`, `TXS_COND_MAP_C` | Verificar TXS_COND_MAP — corrigir ou remover classe customizada com erro |

## Ferramentas de Diagnostico

| Ferramenta | Como Usar |
|-----------|----------|
| **SLG1** (Application Log) | `CL_TXS_LOG_DAO` grava: exception + payload JSON + response JSON + Correlation ID |
| **X-CorrelationID** | UUID do payload — rastrear request no Tax Engine. Mesmo UUID no SLG1 |
| **GLO_LOG_BR_EXT_JSON_RESP** | Feature Toggle — ativa log detalhado do JSON response |
| **Breakpoints ADT** | Colocar nas classes/metodos acima e executar a transacao |

## Sequencia Recomendada de Debug

```
1. CL_TXS_SWITCH::IS_ACTIVE          → TXS esta ativo?
2. CL_TXS_PAYLOAD_TYPE::DETERMINE    → Tipo de payload correto?
3. CL_TXS_COMMUNICATION::POST        → HTTP foi bem-sucedido? Status 200?
4. CL_TXS_RESPONSE_BUFFER::GET       → Response esta no buffer?
5. CL_TXS_CONDITIONS_MAPPER_BR::MAP  → Mapeamento correto para KOMV?
6. CL_TXS_TAX_CODE_DAO::GET          → Tax code mapeado corretamente?
```

## Segundo Breakpoint (Aprofundamento)

| Cenario | Segundo Breakpoint | Classe | Metodo |
