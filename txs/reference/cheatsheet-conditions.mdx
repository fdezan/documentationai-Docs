## Service Call — A Condition que Dispara o HTTP

| Condition Type | Internal Code | Papel | Fonte |
|:-:|---|---|---|
| **TXSC** | CALCULATE_TAX | **Dispara chamada HTTP** ao Tax Engine. Armazena response no buffer singleton. | `if_txs_constants.abap:146` |

<Callout kind="info">
**TXSC deve ser a primeira** condition type TXS na pricing procedure (menor sequencia).
As demais conditions **leem do buffer** — sem HTTP adicional.
</Callout>


## Condition Types por Imposto

## Regras Criticas

1. **Todas** as condition types TXS usam **Formula 279** (requirement formula BR)
2. **TXSC deve ser a primeira** condition type TXS na pricing procedure (menor sequencia)
3. TXSC e a **unica** que dispara a chamada HTTP ao Tax Engine
4. As demais conditions **leem do Response Buffer** (singleton) — sem HTTP adicional
5. O mapeamento condition_type ↔ campo KOMV e feito pela tabela **J_1BNFTXCOND**
6. O mapeamento internal_code → condition_type (KSCHL) e feito pela tabela **J_1BTXCOND**
7. Se TXSC nao executar, todas as demais conditions falham (`CX_TXS_RESPONSE_NOT_ON_BUFFER`)

## Mapeamento — Campos J_1BNFTXCOND → KOMV

Cada condition type mapeia para um campo diferente da KOMV, conforme o campo na J_1BNFTXCOND:

| Campo J_1BNFTXCOND | Metodo do Mapper | Campo KOMV | Fonte |
|---|---|---|---|
| rate | MAP_COND_RATE | KWERT, KBETR = rate × 10 | `cl_txs_conditions_mapper_br.abap:460` |
| value | MAP_COND_AMOUNT | KWERT, KBETR, KAWRT | `:462` |
| base | MAP_COND_BASE | KWERT = taxable | `:464` |
| basered1 | MAP_COND_BASE_REDUCTION_RATE | KWERT, KBETR | `:466` |
| excbas | MAP_COND_EXCLUDED_BASE | KWERT = exempted_base_amount | `:468` |
| othbas | MAP_COND_OTHER_BASE | KWERT = other_base_amount | `:470` |
| whtcollcode | MAP_COND_WHT_COLLECTION_CODE | KNUMA_BO | `:472` |
| srate | MAP_COND_ST_SURCHARGE_RATE | KWERT, KBETR | `:474` |
| qrate | MAP_COND_PAUTA_RATE | KWERT, KMEIN, KPEIN | `:476` |
| qbase | MAP_COND_PAUTA_BASE | KWERT = pauta × 10 | `:482` |
| *(sem match)* | — | KINAK = 'X' | Condition inativada |

### Classe responsavel pelo mapeamento

`CL_TXS_CONDITIONS_MAPPER_BR::MAP_TAX_TYPE_TO_CONDITION`

### Tax code (MWSKZ)

O tax code retornado pelo Engine (`tax_line.taxcode`) e convertido para o tax code SAP (MWSKZ) via tabela `TXS_TAX_CODE_MAP` (classe `CL_TXS_TAX_CODE_DAO`). Variante customizing `TXS_TAX_CODE_MAP_C` tem prioridade.
