## DIA 1 (Seg) — Fundamentos e Arquitetura

**Tema**: *"Por que um Tax Engine externo? Entendendo o TXS de ponta a ponta"*

### Objetivos de Aprendizagem
- Compreender a motivacao para engines externos no S/4HANA Public Cloud (Clean Core)
- Conhecer a arquitetura das 8 fases do TXS
- Entender o principio: N chamadas do pricing, mas apenas 1 chamada HTTP
- Saber quais dados o SAP envia (payload) e recebe (response)
- Conhecer os design patterns usados (Singleton, Factory, Country Specialization)

### Agenda

| Horario | Duracao | Topico | Material |
|---------|---------|--------|----------|
| 19:00-19:30 | 30 min | **Abertura e Contexto** | Slide deck |
| 19:30-20:15 | 45 min | **Arquitetura das 8 Fases** | Diagrama E2E simplificado |
| 20:15-20:25 | 10 min | *Intervalo* | |
| 20:25-21:00 | 35 min | **Payload e Response — Estrutura de Dados** | Diagramas request/response |
| 21:00-21:20 | 20 min | **Design Patterns e Hierarquia de Classes** | Diagrama class hierarchy |
| 21:20-21:30 | 10 min | **Quiz rapido + Preview Dia 2** | |

### Detalhamento

**Abertura e Contexto (30 min)**
- Calculo de impostos: ECC (TAXBRA + condition records locais via FTXP/OBYZ) vs. S/4HANA Cloud (Tax Service externo)
- Motivacao para engine externo:
  - Clean Core: nenhuma modificacao no standard SAP
  - Complexidade tributaria BR: 9+ impostos, ICMS por UF, ST, DIFAL, regimes especiais
  - Atualizacao automatica de aliquotas pelo parceiro fiscal
  - Compliance continuo sem customizacao SAP
- Impostos cobertos: ICMS, ICMS-ST, IPI, PIS, COFINS, ISS, IRRF, CSLL, CBS
- Parceiros de mercado: Sovos, Vertex, Thomson Reuters

**Arquitetura das 8 Fases (45 min)**

| Fase | Nome | O que faz | Classe principal |
|------|------|-----------|-----------------|
| 1 | Entrada e Validacoes | Recebe KOMK/KOMP/KOMV, enriquece LFD, valida ativacao | CL_TXS_CONTROLLER, CL_TXS_SWITCH |
| 2 | Decisao | Service call (1a condition, HTTP) vs. mapping (2a+, buffer) | CL_TXS_CONTROLLER |
| 3 | Build Payload | Constroi JSON: header, items, locations, party, classifications | CL_TXS_PAYLOAD_FACTORY, CL_TXS_ITEMS_BR |
| 4 | Comunicacao HTTP | OAuth2 Client Credentials, JSON camelCase, POST ao engine | CL_TXS_COMMUNICATION, CL_TXS_CONNECTION |
| 5 | Processamento da Resposta | Deserializa JSON, parseia tax_lines/tax_values por tipo | CL_TXS_RESPONSE_BR, CL_TXS_TAX_TYPES_VALUES_BR |
| 6 | Mapeamento KOMV | tax_type → condition types via J_1BNFTXCOND (ICMS→BX16/ICM1, IPI→BIP0/IPI1...) | CL_TXS_CONDITIONS_MAPPER_BR |
| 7 | Verificacao | Pelo menos 1 condition mapeou com sucesso? Se nao: erro | CL_TXS_CONDITION_PROCESSING |
| 8 | Pos-Processamento | BAdI hooks, error handling, reset KOMP | TXS_BADI_CONTROLLER |

Conceito-chave: **Formula 279** dispara o TXS. O pricing chama N vezes (1 por condition type), mas a chamada HTTP acontece apenas 1 vez (TXSC = service call). Demais conditions leem do Response Buffer.

**Payload e Response (35 min)**
- Payload: header (UUID, data, empresa, moeda), items (NCM, CEST, CFOP, preco, qtd), locations (ship_from/ship_to), party (CNPJ, IE, CNAE, SUFRAMA)
- Response: tax_lines[] → tax_values[] (tax_type_code, rate, taxable, value) + tax_attributes (CST, CENQ, CBENEF)

**Design Patterns (20 min)**
- Country Specialization: CL_TXS_* → CL_TXS_*_BR
- Singleton: factories, communication, buffer
- Factory: 3 niveis (enhanced → country → base)
- Strategy: V1/V2/DirectCalc

**Quiz (10 min)**: 5 perguntas rapidas

## DIA 2 (Ter) — Payload, Dados Mestres e Fluxo MM

**Tema**: *"O que o SAP envia ao motor externo e de onde vem cada dado"*

### Objetivos de Aprendizagem
- Dominar a anatomia completa do payload
- Saber de onde vem cada campo (tabela SAP, transacao, app Fiori)
- Conhecer os dados mestres criticos e como valida-los
- Entender o fluxo MM completo (PO, GR, Invoice Verification)

### Agenda

| Horario | Duracao | Topico | Material |
|---------|---------|--------|----------|
| 19:00-19:40 | 40 min | **Anatomia do Payload: Header, Items, Classifications** | Field mapping |
| 19:40-20:15 | 35 min | **Locations, Party e Dados Fiscais BR** | Field mapping |
| 20:15-20:25 | 10 min | *Intervalo* | |
| 20:25-21:00 | 35 min | **Fluxo MM: PO, GR, Invoice Verification** | Diagrama request |
| 21:00-21:15 | 15 min | **Exercicio: Rastreie o Campo** | Handout |
| 21:15-21:30 | 15 min | **Checklist de Dados Mestres + Recap** | Handout |

### Detalhamento

**Anatomia do Payload (40 min)**
- Header: UUID, data, empresa, moeda, operacao
- Items: quantidade, preco unitario, tax code
- Classifications: NCM (MARC-STEUC, 8 digitos sem pontos), CEST, EAN, ExTIPI, NBS
- Additional Info: CFOP, material origin (J_1BMATORG), material usage
- Cost Information: frete, seguro, desconto

**Locations, Party e Dados Fiscais (35 min)**
- Locations MM: ship_from = fornecedor (LFA1), ship_to = planta (T001W)
- Locations SD: ship_from = empresa/planta, ship_to = cliente (INVERSO do MM!)
- Party: CNPJ, CPF, IE, IM, IBGE, SUFRAMA
- CNAE: 8 metodos BRF+ com fallback LFA1-CNAE / J_1BBRANCH-CNAE
- Contribuintes: 7 tipos em J_1BBRANCH

**Fluxo MM (35 min)**
- PO → GR → Invoice Verification (TXS chamado em cada etapa)
- BAdIs de entrada, operation nature, cenarios especiais (STO, importacao, frete)

**Exercicio: Rastreie o Campo (15 min)**: 5 campos → tabela → transacao → impacto

**Checklist de Dados Mestres (15 min)**: Material, BP, Branch, Plant

## DIA 3 (Qua) — Configuracao, Resposta e Fluxo SD

**Tema**: *"Configurando o TXS e entendendo o caminho de volta (motor → KOMV)"*

### Objetivos de Aprendizagem
- Dominar todas as tabelas de configuracao
- Compreender o setup de comunicacao (OAuth2, Logical Port)
- Entender como a resposta vira KOMV (mapeamento reverso)
- Conhecer o fluxo SD completo

### Agenda

| Horario | Duracao | Topico | Material |
|---------|---------|--------|----------|
| 19:00-19:40 | 40 min | **Ativacao e Tabelas de Configuracao** | Handout config |
| 19:40-20:15 | 35 min | **Resposta → KOMV: Mapeamento Reverso** | Diagrama response |
| 20:15-20:25 | 10 min | *Intervalo* | |
| 20:25-21:00 | 35 min | **Fluxo SD: Sales Order, Billing, Delivery** | |
| 21:00-21:20 | 20 min | **Exercicio: Monte a Configuracao** | Handout exercicio |
| 21:20-21:30 | 10 min | **Recap + Preview Dia 4** | |

### Detalhamento

**Ativacao e Configuracao (40 min)**
- CL_TXS_SWITCH: 5 checks (conexao, TXS_ACTIVATION, TXS_ACTIVATION_EXCEPTION, V2 Control, Override)
- TXS_TAX_CODE_MAP: tax code externo → MWSKZ S/4 (variante _C tem prioridade)
- TXS_COND_MAP: condition type → classe mapper
- Communication Arrangement: Logical Port, OAuth2, URL
- Pricing Procedure: TXSC (service call) primeiro + multiplas conditions por imposto (ex: BX16/ICM1 para ICMS, BIP0/IPI1 para IPI, IPIS/BPI1 para PIS, ICOF/BCO1 para COFINS), todos com Formula 279. Mapeamento configurado via J_1BTXCOND.

**Mapeamento Reverso (35 min)**
- JSON → CL_TXS_JSON_SERIALIZER → CL_TXS_RESPONSE_BR → CL_TXS_TAX_TYPES_VALUES_BR → MAPPED_TAX_TYPES[]
- KWERT = value, KBETR = rate x 10, KAWRT = taxable, KINAK = 'X' se sem match

**Fluxo SD (35 min)**
- SO → Delivery → Billing, locations invertidas vs MM
- V1 vs V2: parties no item, _extensions[] YY1_*, item_dates
- Operation Nature: 14+ mapeamentos
- Cenarios: free goods, SUFRAMA, consignment

**Exercicio (20 min)**: 3 empresas, pricing TXBRA1, preencher TXS_ACTIVATION + TXS_TAX_CODE_MAP

### Top 5 Problemas

| # | Problema | Classe | Metodo | Verificar |
|---|---------|--------|--------|-----------|
| 1 | TXS nao chamado | CL_TXS_SWITCH | IS_ACTIVE | TXS_ACTIVATION? Exception? |
| 2 | ICMS zero/errado | CL_TXS_CONDITIONS_MAPPER_BR | MAP_TAX_TYPE_TO_CONDITION | tax_type na response? UFs? |
| 3 | Tax code invalido | CL_TXS_TAX_CODE_DAO | GET_S4_TAX_CODE_SINGLE | TXS_TAX_CODE_MAP? |
| 4 | NCM vazio | CL_TXS_ITEM_CLASSIFICATIONS_BR | GET_MATERIAL_NCM | MARC-STEUC? |
| 5 | HTTP timeout | CL_TXS_CONNECTION | GET_REST_CLIENT | OAuth2? Firewall? |

## Dicas para o Instrutor

1. **Dia 1 e conceitual** — Nao aprofundar em configuracao. Foco em "por que" e "como funciona"
2. **Dia 2 e sobre dados** — Enfatizar que 80% dos problemas vem de dados mestres incompletos
3. **Dia 3 e sobre configuracao** — Mostrar tabelas reais no sistema se possivel
4. **Dia 4 e pratico** — Maximizar exercicios, deixar alunos diagnosticarem antes de revelar respostas
5. **X-CorrelationID** e a ponte entre SAP e motor externo — ensinar a rastrear
6. **BAdI Released** (BD_GSLOG_EXT_TAX_CALC_PAYLOAD) e o unico ponto seguro para Cloud — repetir isso
7. O **cartao de referencia rapida** deve ser distribuido no Dia 4 como material de consulta permanente
