## Visao Geral

No S/4HANA Cloud, a determinacao de impostos brasileiros segue o modelo de Tax Calculation integrado, com especificidades para a legislacao brasileira.

## Condition Technique para Impostos BR

### Estrutura
```
Tax Procedure (TAXBRA / TAXBRAJ)
  └── Condition Types (ICMS, IPI, PIS, COFINS, etc.)
       └── Access Sequences
            └── Condition Tables
                 └── Condition Records (aliquotas, bases, reducoes)
```

### Tax Procedure TAXBRA
Procedimento de calculo de impostos para Brasil. Define:
- Sequencia de calculo dos impostos
- Dependencias entre impostos (ex: IPI na base do ICMS)
- Formulas de calculo
- From/To steps para subtotais

## Fluxo de Determinacao de Impostos

```
1. Documento de Origem (SO, PO, Invoice)
         ↓
2. Determinacao de Tax Code
   - Material + Planta + UF origem + UF destino + Tipo operacao
         ↓
3. Determinacao de CFOP
   - Tipo de operacao + UF origem + UF destino + Categoria fiscal
         ↓
4. Determinacao de CST/CSOSN
   - Por imposto: Origem + Tributacao
         ↓
5. Calculo de Base de Calculo
   - Valor do item +/- deducoes
   - Reducao de base (se aplicavel)
   - IPI na base do ICMS (dependendo da operacao)
         ↓
6. Aplicacao de Aliquota
   - Aliquota por UF, NCM, CFOP, etc.
         ↓
7. Calculo do Imposto
   - Base × Aliquota = Valor do imposto
   - ICMS-ST: (Base ST × Aliquota interna) - ICMS proprio
   - PIS/COFINS: Cumulativo vs Nao-cumulativo
         ↓
8. Geracao da Nota Fiscal com impostos calculados
```

## Particularidades S/4HANA Cloud

### Tax Calculation no Cloud
- Utiliza **Tax Service** integrado ou **SAP Document and Reporting Compliance**
- Configuracao simplificada via Fiori Apps
- Tax Codes determinados automaticamente com base em regras

### Fiori Apps de Tax LOCBR
| App | Descricao |
|-----|-----------|
| Manage Tax Codes | Gerenciar codigos de imposto |
| Manage Tax Rates | Gerenciar aliquotas |
| Manage Tax Determination Rules | Regras de determinacao |
| Monitor Tax Calculation | Monitorar calculos |

### Diferencas vs ECC
- Sem transacoes FTXP, OBYZ, OBCL tradicionais
- Configuracao via Fiori Apps
- Tax Procedure pode ser diferente (TAXBRAJ)
- Integracao com Document Compliance para NF-e
- Calculo pode ser delegado a engine externa (SAP DRC)
