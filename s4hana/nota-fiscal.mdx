## Visao Geral

No S/4HANA Public Cloud, o processamento de Nota Fiscal Eletronica (NF-e) e integrado com o **SAP Document and Reporting Compliance (DRC)** ou solucao equivalente para transmissao SEFAZ.

## Tipos de Documento Fiscal

| Tipo | Descricao | Modelo |
|------|-----------|--------|
| NF-e | Nota Fiscal Eletronica (mercadorias) | Modelo 55 |
| NFC-e | Nota Fiscal de Consumidor Eletronica | Modelo 65 |
| NFS-e | Nota Fiscal de Servico Eletronica | Municipal |
| CT-e | Conhecimento de Transporte Eletronico | Modelo 57 |
| MDF-e | Manifesto Eletronico de Documentos Fiscais | Modelo 58 |

## Fluxo NF-e no S/4HANA Cloud

```
1. DOCUMENTO DE ORIGEM
   Sales Order / Billing Document / Goods Movement / Invoice
              ↓
2. CRIACAO DA NOTA FISCAL
   - Determinacao de impostos (Tax Determination)
   - Determinacao de CFOP
   - Determinacao de CST
   - Calculo de impostos
   - Preenchimento de dados adicionais (frete, seguro, etc.)
              ↓
3. GERACAO DO XML
   - Montagem do XML conforme layout SEFAZ
   - Validacao de campos obrigatorios
   - Calculo de digitos verificadores
              ↓
4. ASSINATURA DIGITAL
   - Certificado A1 ou A3
   - Assinatura do XML com certificado da empresa
              ↓
5. TRANSMISSAO SEFAZ
   - Envio do XML assinado para SEFAZ autorizadora
   - Via SAP Document and Reporting Compliance (DRC)
   - Ou via provedor de comunicacao terceiro
              ↓
6. RETORNO SEFAZ
   - Autorizacao (protocolo de autorizacao)
   - Rejeicao (codigo e mensagem de erro)
   - Denegacao
              ↓
7. POS-PROCESSAMENTO
   - Se autorizada: gerar DANFE, atualizar status, contabilizar
   - Se rejeitada: corrigir e retransmitir
   - Cancelamento (se necessario, dentro do prazo)
   - Carta de Correcao (CC-e)
   - Inutilizacao de numeracao
```

## SAP Document and Reporting Compliance (DRC)

### O que e
Solucao SAP para compliance fiscal eletronico, integrada ao S/4HANA Cloud. Gerencia:
- Geracao de documentos fiscais eletronicos
- Comunicacao com autoridades fiscais (SEFAZ)
- Armazenamento de documentos
- Reporting fiscal (SPED)

### Componentes
- **Document Compliance**: NF-e, CT-e, NFS-e
- **Tax Reporting**: SPED Fiscal, SPED Contribuicoes, ECD, ECF
- **Withholding Tax**: Retencoes (IRRF, CSLL, INSS, PIS, COFINS)

## Estrutura de Dados NF no S/4HANA Cloud

### Entidades Principais
- **BR_NF Document Header**: Cabecalho da NF (numero, serie, data, emitente, destinatario)
- **BR_NF Document Item**: Itens da NF (material, quantidade, valores, impostos)
- **BR_NF Tax**: Impostos por item (ICMS, IPI, PIS, COFINS, etc.)
- **BR_NF Additional Data**: Dados complementares
- **BR_NF Referenced Document**: Documentos referenciados
- **BR_NF Transport**: Dados de transporte
- **BR_NF Payment**: Dados de pagamento

### CDS Views LOCBR (verificar disponibilidade Released)
| CDS View | Descricao |
|----------|-----------|
| I_BR_NFDocument | Cabecalho NF |
| I_BR_NFItem | Item NF |
| I_BR_NFTax | Impostos NF |
| I_BR_NFStatus | Status NF |

## Cenarios de Nota Fiscal

### NF de Saida (Venda)
```
Sales Order → Delivery → Billing Document → Nota Fiscal de Saida
```
- CFOP: 5.XXX (estadual) / 6.XXX (interestadual) / 7.XXX (exportacao)
- Impostos: ICMS, IPI, PIS, COFINS (+ ST se aplicavel)

### NF de Entrada (Compra)
```
Purchase Order → Goods Receipt → Invoice Verification → Nota Fiscal de Entrada
```
- CFOP: 1.XXX (estadual) / 2.XXX (interestadual) / 3.XXX (importacao)
- Impostos: ICMS (credito), IPI (credito), PIS (credito), COFINS (credito)

### NF de Transferencia
```
Stock Transfer Order → Goods Movement → Nota Fiscal
```
- CFOP: 5.151/6.151 (transferencia de producao), 5.152/6.152 (transferencia de mercadoria)

### NF de Devolucao
```
Customer Return → Return Delivery → Credit Memo → Nota Fiscal de Devolucao
```
- CFOP: 1.201/2.201 (devolucao de venda), 5.201/6.201 (devolucao de compra)

### NF Complementar
- Complemento de ICMS, IPI ou preco
- Referencia NF original

## Extensibilidade NF no S/4HANA Cloud

### BAdIs para NF
- BAdIs de determinacao de impostos
- BAdIs de geracao de XML
- BAdIs de validacao de NF
- BAdIs de pos-processamento

### Custom Fields em NF
- Campos adicionais no cabecalho ou item da NF
- Configuravel via Key User Extensibility
- Pode ser incluido no XML via BAdI

### Integracoes
- EDI: Integracao com sistemas de comunicacao SEFAZ
- Arquivamento: Armazenamento de XML e DANFE
- Contabilidade: Lancamentos contabeis automaticos
- Estoque: Movimentacoes de estoque vinculadas

## DANFE - Documento Auxiliar

- Gerado apos autorizacao da NF-e
- Formato PDF
- Contem chave de acesso de 44 digitos
- Codigo de barras para consulta
- No S/4HANA Cloud: gerado via Output Management ou DRC
