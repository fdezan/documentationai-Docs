## Visao Geral

Key User Extensibility permite que usuarios-chave (nao desenvolvedores) estendam o sistema sem codigo ABAP, usando Fiori Apps de extensibilidade.

## Tipos de Extensibilidade Key User

### 1. Custom Fields

**O que e**: Adicionar campos customizados a objetos standard SAP.

**Fiori App**: "Custom Fields" (F1481)

**Onde pode adicionar campos**:
- Business Partner
- Sales Order / Sales Order Item
- Purchase Order / Purchase Order Item
- Billing Document
- Material
- Delivery
- Nota Fiscal (LOCBR)
- E outros objetos habilitados pela SAP

**Caracteristicas**:
- Campo aparece automaticamente nas UIs Fiori habilitadas
- Pode ser usado em filtros, ordenacao e agrupamento
- Disponivel em CDS Views de extensao
- Pode ter logica customizada associada (Custom Logic)
- Transportavel entre ambientes (DEV → QAS → PRD)

**Limitacoes**:
- Apenas em objetos que a SAP habilitou para extensao
- Tipos de campo limitados (texto, numero, data, booleano, value help)
- Maximo de campos por objeto definido pela SAP

### 2. Custom Logic

**O que e**: Logica de negocio customizada via BAdIs Key User, sem codigo ABAP completo.

**Fiori App**: "Custom Logic" (F2680)

**Quando usar**:
- Validacoes customizadas (ex: verificar campo obrigatorio)
- Determinacoes (ex: preencher campo automaticamente)
- Calculos (ex: desconto especial)
- Restricoes (ex: bloquear criacao sob condicao)

**Pontos de extensao disponiveis** (exemplos):
- Before Save / After Save
- Before Create / After Create
- Before Modify / After Modify
- Determination on Modify / on Save
- Validation on Save
- Side Effects

**Linguagem**: ABAP restrito (subset ainda menor que ABAP Cloud)
- Apenas operacoes simples
- Sem acesso a DB direto
- Sem chamadas externas
- Variaveis de contexto fornecidas pelo framework

### 3. Custom CDS Views

**O que e**: CDS Views customizadas sobre dados standard.

**Fiori App**: "Custom CDS Views" (F2170)

**Uso**:
- Criar views para reports customizados
- Fontes de dados para Analytical Queries
- Value Helps customizados
- Combinacao de dados de multiplas fontes

**Caracteristicas**:
- Baseadas em CDS Views Released da SAP
- Suportam joins, filtros, campos calculados
- Podem incluir Custom Fields
- Podem ser consumidas por Custom Analytical Queries

### 4. Custom Business Objects

**O que e**: Objetos de negocio inteiramente novos com UI Fiori gerada.

**Fiori App**: "Custom Business Objects" (F1470)

**Quando usar**:
- Dados que nao existem no standard SAP
- Tabelas auxiliares / cadastros customizados
- Listas de controle, parametrizacoes
- Extensoes de dados que nao se encaixam em Custom Fields

**Caracteristicas**:
- Define estrutura (campos e tipos)
- UI Fiori gerada automaticamente (List Report + Object Page)
- CRUD automatico
- Pode ter Custom Logic associada
- Pode ter associacoes com objetos standard
- Pode ser consumido por APIs

### 5. Custom Analytical Queries

**O que e**: Queries analiticas para SAC (SAP Analytics Cloud) ou Fiori Analytical Apps.

**Fiori App**: "Custom Analytical Queries" (F2096)

**Uso**:
- Reports analiticos customizados
- KPIs
- Dashboards no SAC
- Embedded Analytics no Fiori

## Fluxo de Trabalho Key User

```
1. Identificar necessidade de extensao
          ↓
2. Verificar se objeto suporta extensao Key User
          ↓
3. Criar extensao no ambiente DEV via Fiori App
   (Custom Field / Custom Logic / Custom CDS / Custom BO)
          ↓
4. Testar no DEV
          ↓
5. Transportar para QAS via "Export Customizing Transports" (Fiori App)
          ↓
6. Testar no QAS
          ↓
7. Transportar para PRD
```

## Limitacoes Key User Extensibility

- Apenas objetos habilitados pela SAP podem ser estendidos
- Logica customizada tem ABAP muito restrito
- Sem acesso a APIs externas diretamente
- Performance pode ser impactada com muitos Custom Fields
- Numero maximo de extensoes por objeto
- Nem toda UI Fiori suporta Custom Fields automaticamente
